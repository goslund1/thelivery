### Step 3 — Rust compositor (`backend/src/compositor.rs`)

Pure-Rust image compositor using `fontdue` (TTF rasterizer) and the `image` crate. Fonts embedded with `include_bytes!`:

- **BebasNeue-Regular.ttf** — via jsDelivr CDN (`cdn.jsdelivr.net/gh/google/fonts@main/...`)
- **Oswald-VF.ttf** — variable font via `raw.githubusercontent.com/google/fonts/main/ofl/oswald/Oswald%5Bwght%5D.ttf`

Both are OFL (SIL Open Font License) — safe to bundle and redistribute.

**Text pipeline:** `measure_text()` → `rasterize_text_box()` (renders at natural height, Lanczos3-resizes to fill box w×h, post-resize alpha clamp to fix sinc ringing) → `blit_transformed()` (inverse pixel mapping for rotation + shear, Porter-Duff `alpha_over`).

| POSTCARD | Bebas Neue | Floating white, full alpha, drop shadow |
| SIGNAL | Oswald VF | Dark backdrop plate (chyron), full alpha, drop shadow |
| GHOST | Oswald VF | Semi-transparent (170/255), no shadow, delicate |

### Step 6 — Compositor polish (this session)

**Rust additions to `compositor.rs`:**

- **`apply_bottom_scrim`** — quadratic dark gradient (y=45%→100%, max 160/255 ≈ 63% opacity). Auto-fires when any visible text box is in the lower half. Critical for legibility on bright/light photos.
- **`blit_backdrop`** — 10px-padded dark plate (190/255 opacity) behind SIGNAL text, same rotation+shear as text. Chyron / broadcast lower-third visual identity.
- **`blit_transformed_tinted`** — like `blit_transformed` but replaces pixel colour with a tint, preserving source alpha shape. Used for drop shadows; source alpha × tint alpha keeps glyph edges smooth.
- **Drop shadows** — POSTCARD and SIGNAL get 2×3px offset shadow (150/255 black) via `blit_transformed_tinted`. GHOST intentionally skipped.
- **`rasterize_text_natural`** — natural-width rasterizer (no stretch). Used for fixed-size glyphs.
- **`render_logo`** — "THE LIVERY" in Bebas Neue 22px, bottom-right corner (24px margin), 180/255 opacity + mini shadow. Placeholder until a real logo asset exists; swap in `render_logo` when ready.
- **Min box size guard** — `w_px.max(10)` / `h_px.max(8)` prevents sub-pixel boxes producing invisible output.

## Key architectural decisions

- **Single-renderer architecture** — DOM canvas is interaction-only; the Rust compositor is the sole source of truth. Live preview calls `POST /share/preview` (same code path as public `GET /share/:id/card.png`). No drift between editor and output.
- **Fractional coordinates** — all positions stored as 0.0–1.0 fractions so they scale cleanly to any output resolution.
- **Inverse pixel mapping** — for rotation + shear, the blitter iterates output pixels and back-solves to source coordinates rather than forward-mapping (avoids gaps).
- **Lanczos3 alpha clamp** — sinc ringing at glyph edges can push scaled alpha above the source max. Post-resize per-pixel clamp fixes this (especially visible in GHOST at 170/255).
- **Fonts via include_bytes!** — zero runtime dependency; works identically in dev and prod binary.

## Key bugs / lessons

- **Font 404s** — GitHub raw URLs for Google Fonts returned 404. Bebas Neue works via jsDelivr (`cdn.jsdelivr.net/gh/google/fonts@main/...`). Oswald is a variable font (`Oswald[wght].ttf`), not static variants — static URLs all 404; correct URL uses `%5B` / `%5D` URL-encoding for the brackets.
- **`OgTextBox[]` missing `id`** — boxes arriving from `OgMakerConfig` (preset configs) lack `id`. Fixed init: `cfg.boxes.map(b => ({ ...b, id: (b as any).id ?? crypto.randomUUID() }))`.
- **`photoId: number | null` → `number`** — `buildConfig()` could have null photoId but `OgConfig.photoId` is `number`. Fixed: `photoId: photoId.value ?? 0`.
