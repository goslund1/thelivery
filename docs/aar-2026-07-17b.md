# AAR — 2026-07-17 (b): Full read-through, backend module split, test suite

Second session of the day (first: OG Maker 12-font shakedown). This one was a
deliberate quality pass: a full read-through of the codebase, then a sequenced
cleanup with a commit per item so anything can be walked back independently.

## What was done

1. **Full codebase read-through** (~25k lines: main.rs entire, data model, api,
   stores, App/CardView, RecipeSection head, docs). Verdict: A− code, A+
   process. Structure is stable post-migration-0017; the gap to A+ was
   regression safety, not refactoring.

2. **Stale `variants[]` fix** (`4ab82e7`) — `CardView.vue` `isSectionEmpty` and
   `multiCarDetected` still checked the deprecated `recipe.variants` instead of
   `cars`. Symptom: cards that already had car tabs showed an active-looking
   "Car tabs" button that silently no-op'd.

3. **Backend module split** (`5352d37`) — main.rs (3363 lines) → thin entry
   point (~170 lines) + domain modules along the existing section banners:
   `state`, `auth`, `cards`, `images`, `trash`, `identity`, `share`,
   `suggestions`, `presets`, `theme`. Mechanical extraction (sed by line range,
   blanket `pub`, per-module use headers); verified by cargo build + endpoint
   smoke test.

4. **Light card-body validation** (`294ff5b`) — `validate_card_body` on
   PUT/POST rejects only what would corrupt the catalog (non-object body,
   wrong-typed name/catalogNumber, sections without string type+key, non-array
   images/tags/collections). Unknown fields and unknown section types pass
   through untouched — tests assert this explicitly, preserving the
   "backend never changes for new sections" property.

5. **list_cards N+1 fix** (`bb392e6`) — `fetch_all_images_grouped` pulls all
   image rows in one query, grouped by card_id. Verified **byte-identical**
   output against the still-running pre-change backend on :8787.

6. **theme.ts static import** (`6b72f18`) — its dynamic `import('./ui')` calls
   were cargo-cult copies of the ui↔modal pattern; ui.ts never imports
   theme.ts, so there was no cycle. The genuine ui↔modal cycle stays dynamic.

7. **Circular-import verdict SETTLED** (`90cb269`) — reviewed the cycle a
   second time at Jason's request. Conclusion: the previous decision was right;
   a signal-module extraction only fixes one direction and would scatter
   post-login flow into LoginModal.vue. Canonical rationale now lives in
   `ui.ts → handleAuthError`, pointed to from modal.ts, plus a frontend-gotchas
   entry so the INEFFECTIVE_DYNAMIC_IMPORT build warning never triggers a third
   investigation.

8. **`.env` gitignored** (same commit) — was untracked but NOT ignored; one
   `git add -A` away from committing ANTHROPIC_API_KEY. `.env` / `.env.*`
   added next to the users.json secret rule.

9. **Migration/transform test suite** (`ab5a80c`) — 24 new tests (29 total,
   ~0.1s): `normalize_card`, `ensure_standard_sections`,
   `migrate_variants_to_cars` (idempotency = run twice, assert byte-identical),
   `sync_card_images` all three branches + COALESCE preservation against
   in-memory SQLite, end-to-end `normalize_bodies`, `slugify`/`card_folder`
   tables. `testutil::test_pool()`: max_connections(1) is load-bearing —
   each `sqlite::memory:` connection is its own empty DB.

10. **CLAUDE.md brought current** (`5bf1d53`) — backend gate is now
    `cargo test` (was `cargo build`; "no test suite" claim removed), module
    layout documented, idempotency asserts noted as the house pattern.

11. **Docs housekeeping** (this commit) — 21 finished plan/handoff/design/AAR/
    audit files moved to `docs/completed/`; architecture.md backend file map
    updated (was "entire backend, ~2900 lines", migrations 0001–0013).

## Lessons

- **Test-writing surfaced real behavior immediately**: the first draft of the
  normalize_bodies test wrongly assumed body images kept string ids — step 3
  replaces them with integer DB PKs and strips paths. The corrected test now
  documents that. The one dev-loop failure was the FK constraint correctly
  rejecting a dangling livery_id fixture.
- **"Byte-identical against the running old backend" is the gold-standard
  refactor check** when a pre-change process is still up — diff the two ports.
- **Re-examining a settled decision is fine once; then write it down.** The
  SETTLED comment + gotchas entry exist so the same warning doesn't cost a
  third conversation.
