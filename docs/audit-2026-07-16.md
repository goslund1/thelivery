## Correctness

### Scrim fires on top-edge, not box overlap ŌĆö compositor.rs:43
`tb.y > 0.4` checks the box's *top* edge. A box at `y:0.3, h:0.5` (top 30%, bottom 80%)
doesn't trigger the scrim even though most of its content is in the lower half.
Fix: `tb.y + tb.h > 0.45`. Only affects non-default placements; default spawn is at y:0.72.
**Worth fixing.**

### Content edit fires preview on blur, not keystroke ŌĆö OgMakerModal.vue:378
Uses `@change` (fires on blur). Rotate/shear sliders fire on each value change via
`v-model.number`, but typed content changes don't update the preview until focus leaves
the field. Fix: `@change` ŌåÆ `@input`. **Should fix.**

## Architecture / Consistency

### `list_og_presets` is unauthenticated ŌĆö main.rs:1611
`GET /api/og-presets` has no `_auth: AuthUser` extractor; POST/PUT/DELETE all require auth.
ShareModal's `fetchPresets()` calls without auth headers (it's inside `v-if="auth.isAuthenticated"`
so only authenticated users see the UI). Data is non-sensitive today, but any future
field added to presets would be inadvertently public. Fix: add `_auth: AuthUser` and the
auth header to `fetchPresets()`. **Worth fixing.**

### Unnecessary `as any` in `firstCarName` ŌĆö ShareModal.vue:29
After `'cars' in r`, TypeScript should narrow `r` to `ForzaRecipeSection` (the only Section
type with a `cars` field). `(r as any).cars?.[0]?.carName` can be `r.cars?.[0]?.carName`
or `(r as ForzaRecipeSection).cars?.[0]?.carName`. **Worth fixing.**
